Energy source

The energy source component is used to model the generation or import of a commodity inside the energy system.

The following attributes are available:

pydantic model ensysmod.schemas.EnergySourceCreate[source]

Attributes to receive via API on creation of an energy source.

field capacity_per_plant_unit: Optional[float] = None

Capacity per plant unit. By default is 1, thus the number of plants is the equal to the installed capacity.

field capacity_variable: Optional[bool] = None

Whether the energy component should be model with a capacity or not.

field capacity_variable_domain: Optional[CapacityVariableDomain] = None

Mathematical domain of the capacity variables.’continuous’ means that the capacity is modeled as real values >= 0.’discrete’ means that the capacity is modeled as integer values >= 0.

field commodity: str [Required]

Commodity the energy source is associated with.

Constraints

  • example = electricity

field commodity_cost: Optional[float] = None

Cost of the energy source per unit of energy.

Constraints

  • example = 42.2

field description: Optional[str] = None

Description of the energy component. Can be used to explain the component.

field economic_lifetime: Optional[int] = None

Economic lifetime.

field interest_rate: Optional[float] = None

Interest rate.

field invest_per_capacity: Optional[float] = None

Investment per capacity.

field name: str [Required]

Unique name of the energy component. It is used to identify the component.

Constraints

  • example = gas turbine

field opex_per_capacity: Optional[float] = None

Operational expenditure per capacity.

field ref_dataset: int [Required]

Reference to dataset that energy component belongs to.

Constraints

  • example = 1

field shared_potential_id: Optional[str] = None

Shared potential ID. If specified the maximum potential capacity is shared among all components of the same shared potential id.

field type: EnergyComponentType = EnergyComponentType.SOURCE
